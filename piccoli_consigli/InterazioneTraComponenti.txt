ESP dato lo stato modifica il comportamento del suo led.
ESP comunica al server il valore del sonar
	- Distanza 
	- Timestamp	
	- Stato

Arduino legge i dati del server
	- Dato lo stato, modifica il comportamento del suo led
	- Data l'apertura della diga, modifica l'angolo del servomotore

Arduino legge i dati da bluethoot
	- Modifica l'apertura della diga

APP legge i dati del server
	- modifica i valori della sua interfaccia
		- Stato
		- Apertura diga
		- TimeStamp
		- Distanza (con massimali in rosso/giallo)

APP quando è in stato modifica, invia dati al server
	- Avendo l'ultimo valore dato dal server, lo modifica
	- Una volta modificato, invia dati a bluethoot ed al server

Server
	- gestisce la memorizzazione dei dati in LinkedList
	- Quando lo stato è di allarme, gestisce l'apertura della diga e la comunica ad arduino
	- Quando arduino modifica un valore, lo comunica tramite seriale e il server si occupa di modificare il suo valore nella variabile interna

Server avrà /api/data/esp dove memorizzerà tutti i dati strutturati ricevuti da ESP
	avrà inoltre una variabile interna dove, dopo il calcolo dell'apertura della diga, memorizzerà la sua apertura percentile

Dashboard legge i dati del server e stampa
	- Distanza 
	- Timestamp	
	- Stato
	- Apertura diga (variabile)
